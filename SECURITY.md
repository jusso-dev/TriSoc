# Security policy

Do not report vulnerabilities in a public issue. Until a dedicated security
address is published, use the repository host's private vulnerability reporting
feature. Include impact, reproduction, affected commit, and a safe proof of
concept; never include live credentials or customer evidence.

The foundation release is for local evaluation. Bind MCP to loopback, replace the
development database password, do not mount cloud credential directories into
the container, and do not put unredacted cloud responses in issues or fixtures.

Supported versions and coordinated disclosure timelines will be published with
the first tagged release.
